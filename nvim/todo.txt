📝 TODO: Улучшение работы с LSP Diagnistics (Ошибками)
1. Базовая визуализация (Сделать сейчас)

    Virtual Text: Убедиться, что текст ошибки виден справа от строки кода (уже настроено через vim.diagnostic.config).
    Signs (Значки): Проверить, что слева стоят иконки E (Error), W (Warning). Если они мешают или некрасивые — настроить свои символы или цвета.
    Underline: Подтвердить, что ошибочный код подчеркнут волнистой линией.

2. Навигация и быстрый просмотр (Удобство)

    Jump to Error: Привыкнуть к [d / ]d для прыжка между ошибками.
    Hover on Demand: Использовать <leader>e (vim.diagnostic.open_float) для просмотра полного текста ошибки во всплывающем окне, если она не влезает в virtual text.
    Quickfix List: Настроить маппинг (например, <leader>qf), который скидывает все ошибки текущего файла в стандартное окно Quickfix (vim.diagnostic.setqflist() + copen). Это удобно, чтобы видеть список всех проблем сразу.

3. Продвинутый UI (Рекомендуется установить)

    Trouble.nvim: Установить плагин folke/trouble.nvim
    .
        Зачем: Заменяет скучный список ошибок на красивое дерево с группировкой по файлам/папкам.
        Фишка: Можно кликать по ошибке, и курсор прыгнет в нужное место. Показывает иерархию проекта.
        Маппинг: <leader>xx для открытия глобального списка ошибок.
    Noice.nvim: Рассмотреть плагин folke/noice.nvim
    .
        Зачем: Делает всплывающие окна (hover, diagnostics, messages) очень красивыми, с анимацией и прозрачностью. Убирает визуальный шум.

4. Интеграция с Clangd (Для C++)

    Clang-Tidy Checks: Изучить возможность включения конкретных проверок clang-tidy через .clang-tidy файл в корне проекта, чтобы получать более специфичные предупреждения (например, про производительность или безопасность), а не только синтаксис.
    Inlay Hints: Включить подсказки типов прямо в коде (если clangd поддерживает).
        Как: Добавить в настройки clangd: inlayHints = { enabled = true }. Это покажет типы переменных прямо рядом с ними, как в VS Code.
